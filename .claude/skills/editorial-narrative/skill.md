# Editorial Narrative — 에디토리얼 콘텐츠 생성

## 목적
개별 논문 분석을 넘어서, 이번 주 AI 연구의 큰 흐름을 읽고 독자에게 "왜 이 논문들이 함께 중요한지"를 설명하는 에디토리얼 콘텐츠를 생성한다.

## 워크플로우

### Step 1: 전체 논문 데이터 로드
1. `_workspace/01_scout_papers.json` 읽기 — 메타데이터, tags, estimated_impact
2. `_workspace/02_analyst_*.json` 모두 읽기 — 심층 분석 결과

### Step 2: 크로스-페이퍼 분석
1. 모든 논문의 tags, problem, why_it_matters를 교차 대조
2. 반복되는 키워드 클러스터링 (예: "3D", "실시간", "오픈소스")
3. 공통 문제의식 그룹핑 (예: "효율성", "접근성 민주화")
4. 유사한 방법론 패턴 파악 (예: "확산 모델 활용", "강화학습 정렬")

### Step 3: 메가 트렌드 도출
1. 클러스터에서 2-3개 메가 트렌드 선정
2. 각 트렌드에 해당하는 논문 ID 매핑
3. 모든 논문이 최소 1개 트렌드에 속하는지 확인
4. 트렌드 간 관계 분석 (수렴, 독립, 대립)

### Step 4: 에디터스 픽 선정
1. estimated_impact가 "high"인 논문 중 3개 선정
2. 선정 기준: 독창성, 산업 영향력, 접근성
3. 각 픽에 대해 편집자 관점의 코멘트 작성

### Step 5: 논문 관계 맵 생성
1. 모든 논문 쌍에 대해 관계 유무 판단
2. 관계 유형 분류: complementary, contrasting, builds_on, same_trend
3. 강한 관계만 선택 (약한 연결은 제외)
4. 각 관계에 1-2문장 설명 추가

### Step 6: 읽기 가이드 작성
1. 입문자용: 쉬운 난이도 → 중급으로 자연스러운 흐름
2. 전문가용: 가장 임팩트 있는 논문 중심, 효율적 순서
3. 각 가이드에 추천 이유 명시

### Step 7: JSON 출력
- 모든 결과를 `_workspace/04_editorial.json`에 저장
- editorial-director 에이전트의 출력 JSON 스키마를 따름

## 콘텐츠 작성 원칙

### 트렌드 제목
- 짧고 임팩트 있게 (5-8자)
- 예: "3D의 민주화", "에이전트의 자각", "작지만 강한"
- 너무 추상적이면 안 됨 — 구체적 기술/현상 지칭

### 내러티브 어조
- 에디토리얼 에세이 톤 (지적이되 편안한)
- 1인칭 복수("우리") 또는 3인칭 서술
- 명확한 관점 제시 — "이건 중요하다" vs "이건 한계가 있다"
- 비유와 은유 적극 활용

### 관계 설명
- "A는 B의 문제를 다른 각도에서 해결한다" (complementary)
- "A와 B는 같은 벤치마크에서 경쟁하지만 철학이 다르다" (contrasting)
- "A는 B의 기법을 확장하여 새로운 영역에 적용했다" (builds_on)
- "A와 B 모두 '효율성'이라는 주간 메가 트렌드의 일부다" (same_trend)

## 품질 기준
- 트렌드 3개 이내 (2-3개가 이상적)
- 에디터스 픽 3개
- 논문 관계 최소 5쌍
- 읽기 가이드 2종 (입문/전문가)
- 모든 논문이 최소 1개 트렌드에 매핑
