# UX Review — 매거진 웹사이트 UX 검토 및 개선

## 목적
생성된 Paper Pulse 매거진 웹사이트의 UX 품질을 검증하고, 발견된 문제를 직접 코드 수정으로 해결한다.

## 워크플로우

### Step 1: 코드 읽기
1. `index.html` 전문 읽기
2. `style.css` 전문 읽기
3. `script.js` 전문 읽기

### Step 2: UX 검증 (체크리스트 기반)

#### 2-1. 에디토리얼 UX (최우선)
- 트렌드 섹션이 히어로 아래에 배치되어 있는가
- 에디터스 픽 이유가 Featured 카드에 통합되어 있는가
- 논문 관계가 모달 내에서 "관련 논문" 링크로 표현되는가
- 읽기 가이드(추천 순서)가 접근 가능한 위치에 있는가

#### 2-2. 모달 UX
- 이전/다음 논문 네비게이션 존재 여부
- 분석 섹션이 아코디언/탭으로 단계적 공개되는가
- 모달 내 스크롤이 부드러운가
- 닫기 버튼이 항상 접근 가능한가 (sticky)
- 관련 논문 링크 클릭 시 같은 모달 내에서 전환되는가

#### 2-3. 커뮤니티 시그널
- upvote 수 / 출처 뱃지가 카드에 표시되는가
- 시각적 가중치가 적절한가 (주인공은 논문 제목이어야 함)

#### 2-4. 필터 & 검색
- 빈 상태 처리 ("해당 조건의 논문이 없습니다" 메시지)
- 필터 변경 시 부드러운 전환 애니메이션
- 활성 필터가 시각적으로 명확한가

#### 2-5. 접근성
- focus-visible 스타일 존재 여부
- 모달 포커스 트랩 (Tab이 모달 밖으로 나가지 않도록)
- Skip to content 링크
- 이미지 alt 텍스트 품질
- color-only 정보 전달 없이 텍스트/아이콘 병행

#### 2-6. 읽기 추적
- localStorage 기반 "읽음" 표시 구현 여부
- 읽은 카드에 시각적 인디케이터 (체크마크 또는 opacity 변화)

### Step 3: 우선순위 분류

| 우선순위 | 기준 | 행동 |
|---------|------|------|
| **높음** | 사용성에 직접 영향 + 구현 용이 | 즉시 코드 수정 |
| **중간** | UX 개선이지만 필수는 아님 | 가능하면 수정, 아니면 리포트 |
| **낮음** | 나이스투해브 | 리포트만 작성 |

### Step 4: 코드 수정
- 발견된 높음/중간 우선순위 문제를 직접 수정
- 수정 시 기존 디자인 시스템(CSS variables, 색상, 폰트)을 준수
- 새로운 기능 추가 시 기존 코드 패턴을 따름
- 수정 전후를 `_workspace/05_ux_review.md`에 기록

### Step 5: 리뷰 보고서 작성
- `_workspace/05_ux_review.md`에 전체 리뷰 결과 기록
- 수정 완료 항목과 미수정 항목을 분리하여 기록

## 핵심 UX 개선 사항 (반드시 확인)

### 1. 모달 내 논문 네비게이션
```javascript
// 이전/다음 버튼 추가
// 키보드 좌우 화살표로도 이동 가능
```

### 2. 빈 상태 처리
```javascript
// 필터 결과가 0건일 때 안내 메시지 표시
// "조건에 맞는 논문이 없습니다. 필터를 조정해보세요."
```

### 3. 읽기 추적
```javascript
// 모달에서 논문을 열면 localStorage에 기록
// 읽은 카드에 시각적 표시 (opacity 또는 체크 아이콘)
```

### 4. 커뮤니티 시그널 노출
```javascript
// source 필드에서 upvote 수 파싱
// 카드에 작은 뱃지로 표시 (예: "🔥 368")
```

### 5. 관련 논문 링크
```javascript
// editorial의 paper_relations 데이터 활용
// 모달 하단에 "관련 논문" 섹션 추가
// 클릭 시 해당 논문 모달로 전환
```

## 수정하면 안 되는 것
- 전체 디자인 방향/톤 변경
- 색상 시스템 변경
- 폰트 변경
- 데이터 구조 변경
- 기존 잘 작동하는 기능 제거
